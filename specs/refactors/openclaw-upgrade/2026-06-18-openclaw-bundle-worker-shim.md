# OpenClaw bundled gateway worker shim

## 1. 背景

OpenClaw 升级到 `v2026.6.1` 后，Windows bundled gateway fast path 会通过 runtime root 下的 `gateway-bundle.mjs` 启动。

2026-06-17 18:40 附近的 gateway 日志中出现过如下异常：

```text
provider auth state pre-warm failed: Error: Cannot find module ... model-provider-auth.worker.mjs
```

实际 worker 文件位于：

```text
dist/agents/model-provider-auth.worker.js
dist/agents/code-mode.worker.js
dist/agents/compaction-planning.worker.js
```

但 bundled gateway 场景下，OpenClaw 的 worker resolver 会把 fallback 路径推导到 runtime root，例如：

```text
model-provider-auth.worker.mjs
code-mode.worker.mjs
compaction-planning.worker.mjs
```

这些 root-level 文件不存在，因此 provider auth worker 会加载失败。`code-mode` 与 `compaction-planning` 使用同类 resolver，存在相同风险。

## 2. 最终方案

在 LobsterAI 侧生成 root-level worker shim，使 OpenClaw 当前 fallback 路径变为可用路径。

生成文件：

```text
model-provider-auth.worker.mjs
code-mode.worker.mjs
compaction-planning.worker.mjs
```

文件内容只负责 import 真实 worker。例如：

```javascript
// Generated by LobsterAI. Bridges gateway-bundle.mjs worker resolution to dist/agents.
import './dist/agents/model-provider-auth.worker.js';
```

关键点：

- 不复制 `dist/agents/*.worker.js` 到 root，避免破坏 worker 内部相对 import。
- 不修改 OpenClaw 源码。
- 只覆盖带 LobsterAI 生成标记的旧 shim。
- 如果 root 下已有同名非 LobsterAI 文件，则保留并记录 warning。
- 如果 `gateway-bundle.mjs` 不存在，则不生成 shim。
- 如果真实 `dist/agents/*.worker.js` 缺失，则不生成对应 shim 并记录 warning。

## 3. 实施位置

新增共享逻辑：

- `scripts/openclaw-worker-shims.cjs`
- `src/main/libs/openclawWorkerShims.ts`

构建阶段：

- `scripts/bundle-openclaw-gateway.cjs`
- 在 `gateway-bundle.mjs` 已最新并跳过 bundle 时，仍确保 shim 存在。
- 在 `gateway-bundle.mjs` 重新生成成功后，确保 shim 存在。

启动阶段：

- `src/main/libs/openclawEngineManager.ts`
- 在 `ensureBareEntryFiles(runtimeRoot)` 检测到 `gateway-bundle.mjs` 并走 bundled gateway fast path 时，确保 shim 存在。
- 该路径可修复已有本地 runtime，避免为了补 shim 重新编译整个 OpenClaw。

## 4. 打包适用性

Windows：

- OpenClaw runtime 会被打入安装资源。
- 只要 shim 在打包前已生成，就会进入安装包。
- 启动阶段自愈可以覆盖构建或打包路径遗漏 shim 的情况。

macOS/Linux：

- 当前问题主要由 Windows `gateway-bundle.mjs` fast path 触发。
- 统一生成 shim 对 macOS/Linux 无害。
- 如果未来 macOS/Linux 也启用同类 bundled gateway fast path，shim 可提前覆盖相同问题。

## 5. 验证

已新增单元测试：

- `src/main/libs/openclawWorkerShims.test.ts`

覆盖场景：

- 缺少 `gateway-bundle.mjs` 时不生成 shim。
- 真实 `dist/agents/*.worker.js` 存在时生成 root-level shim。
- 已有 LobsterAI 生成的旧 shim 可更新。
- 已有非 LobsterAI 同名文件不会被覆盖。
- 缺少真实 worker target 时会报告并跳过对应 shim。
- CJS helper 与 TS helper 生成内容保持一致。

已执行验证：

```text
npx vitest run src/main/libs/openclawWorkerShims.test.ts src/main/libs/openclawPatches
npx eslint src/main/libs/openclawWorkerShims.ts src/main/libs/openclawWorkerShims.test.ts src/main/libs/openclawEngineManager.ts
npm run compile:electron
```

本验证未重新编译 OpenClaw。

## 6. 风险边界

- shim 列表需要随 OpenClaw 新增同类 worker resolver 时同步更新。
- 如果未来 OpenClaw 正式发布 root-level 同名 worker 文件，LobsterAI 不会覆盖该文件，因为非 LobsterAI shim 会被保护。
- 如果 OpenClaw 改变 worker 文件名、扩展名或 resolver 规则，需要重新核对 shim target。

## 7. 结论

该问题来自 LobsterAI bundled gateway fast path 与 OpenClaw `v2026.6.1` 新增 worker resolver 的组合。

最终采用 LobsterAI 侧 root-level worker shim：

```text
root *.worker.mjs -> import './dist/agents/*.worker.js'
```

该方案改动面小，适合安装包用户，对 macOS/Linux 无害，并且支持启动阶段自愈，无需重新编译整个 OpenClaw。
